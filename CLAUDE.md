## Project Overview

This is a **BMAD v6.8.0** workspace for "Majed's portfolio" — a Next.js portfolio site. The repo is currently in the **planning/design phase**; no application source code exists yet.

BMAD version: 6.8.0  
WDS module: v0.4.3  
CIS module: v0.2.1

## Repository Structure

```
_bmad/               # BMAD framework config (installer-managed, treat as read-only)
  config.toml        # Base project config — regenerated on install, do not edit directly
  config.user.toml   # Personal config (same constraint)
  custom/            # Safe to edit: team overrides (config.toml) and personal overrides (config.user.toml)
_bmad-output/        # Generated artifacts from BMAD agents
  planning-artifacts/
  implementation-artifacts/

  B-Trigger-Map/
  C-UX-Scenarios/
  D-Design-System/
  E-Development/
docs/                # Project knowledge base (referenced by BMM and WDS modules)
.github/agents/      # Agent definition files (loaded by IDE)
.agents/skills/      # Full agent skill implementations
```

## WDS Workflow (current methodology)

The project uses **WDS v6** (Workflow Design System):

1. **Saga** produces `design-artifacts/A-Product-Brief/` and `design-artifacts/B-Trigger-Map/`
2. **Freya** produces `design-artifacts/C-UX-Scenarios/`, `design-artifacts/D-Design-System/`, and `design-artifacts/E-Development/` (Work Orders)
3. **Mimir** reads Freya's Work Orders → writes PRD → implements one verified task at a time

## Configuration

To override BMAD config durably without risking overwrite on reinstall:
- **Team-wide**: edit `_bmad/custom/config.toml`
- **Personal only** (gitignored): edit `_bmad/custom/config.user.toml`

Do **not** edit `_bmad/config.toml` or `_bmad/config.user.toml` directly — both are regenerated by the installer.

## Browser Verification

Use `playwright-cli` as the primary browser automation path for local UI checks, screenshots, snapshots, interaction testing, and Playwright test debugging. If `playwright-cli` is unavailable, blocked, cannot attach to the needed browser/session, or the task specifically needs MCP-native browser tools, fall back to the Playwright MCP browser tools.
